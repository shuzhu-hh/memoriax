# EbbVocab 执行协议

本协议用于确保开发过程稳定、可回退、可部署。

---

## 一、核心原则

1. 一次只做一个 Issue
2. 小步提交
3. 不自动执行命令
4. 所有数据库变更必须通过 Alembic
5. 所有权限必须后端校验

---

## 二、禁止行为

- 手改数据库
- 删除核心配置文件
- 使用危险命令
- 前端实现安全逻辑替代后端

---

## 三、标准流程

开发前：
- git status

修改后：
- git diff
- 说明变更点
- 提示我执行命令

完成后：
- 更新 DEVELOPMENT_PLAN.md
- 建议打 tag

---

## 四、数据库规范

- 所有模型变更必须生成迁移
- 不允许跳过迁移
- 生产必须使用 Postgres

---

## 五、部署前检查

- 使用 Postgres
- 迁移已执行
- CORS 正确
- SECRET_KEY 已设置
- 不同用户数据隔离
